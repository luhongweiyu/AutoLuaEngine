returns: "integer, integer 或 nil 或 nil, string"
---

**方法名称：** 在当前截图中查找模板图片。

**语法：** `findPic(x1, y1, x2, y2, picName, deltaColor, dir, sim)`

**参数说明：**

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `x1` | `integer` | 是 | 查找区域左上或任意一角的横坐标。 |
| `y1` | `integer` | 是 | 查找区域左上或任意一角的纵坐标。 |
| `x2` | `integer` | 是 | 查找区域右下或另一角的横坐标。 |
| `y2` | `integer` | 是 | 查找区域右下或另一角的纵坐标。 |
| `picName` | `string` | 是 | 模板图片路径。可为脚本目录相对路径、绝对路径或 `.alpkg` 资源路径。 |
| `deltaColor` | `string` | 是 | RGB 单通道容差，例如 `"101010"`。`"000000"` 表示完全一致。 |
| `dir` | `integer` | 是 | 扫描方向，取值 `1` 到 `8`，与 `findColors` 的方向约定一致。 |
| `sim` | `number` | 是 | 匹配比例，范围 `(0, 1]`，例如 `0.95`。 |

| 返回值 | 说明 |
|---|---|
| `integer, integer` | 找到时返回模板左上角坐标 `x, y`。 |
| `nil` | 未找到时只返回 `nil`。 |
| `nil, string` | 参数、模板解码或截图失败时返回错误信息。 |

**详细说明：**

```lua
local x, y, errorMessage = findPic(
    0, 0, 1079, 1919,
    "images/start.png",
    "101010",
    2,
    0.95
)

if x then
    print("模板坐标：", x, y)
elseif errorMessage then
    print("找图失败：", errorMessage)
end
```

透明 PNG 的透明像素不参与匹配。模板在 native 内缓存；脚本运行中替换同一路径的模板文件后，
普通文件会按修改时间自动重载，也可调用 `clearImageCache(picName)` 立即清理。模板缓存默认
上限为 `5 MiB`，超限按 LRU 淘汰，脚本结束时全部释放。
