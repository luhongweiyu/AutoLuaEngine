# 回调、线程与生命周期

## 回调参数

| 注册方法 | 回调参数 |
|---|---|
| `setOnClick(handle, callback)` | `function(handle)` |
| `setOnCheck(handle, callback)` | `function(handle, checked)` |
| `setOnSelectEvent(handle, callback)` | `function(handle, index, text)` |
| `setOnSelectEventEx(handle, callback)` | `function(handle, row, column, text)` |
| `setOnSliderEvent(handle, callback)` | `function(handle, position)` |
| `setOnClose(handle, callback)` | `function(handle)`；明确返回 `false` 可取消关闭 |

同一事件重复注册会覆盖旧函数，传入 `nil` 会移除。回调错误写入引擎日志，不会让 OpenGL
渲染线程崩溃。

## `post`

```lua
local accepted = imgui.post(function()
    imgui.setInputText(input, "异步更新完成")
end)
```

`imgui.post(callback)` 返回 `boolean`。它把一次性函数标识送入同一个事件泵，函数执行前
不会进入 Android UI 线程或 OpenGL 线程。投递失败时函数引用立即释放。

## 线程模型

```text
Android 主线程       WindowManager、Surface、触摸和输入法转发
OpenGL 渲染线程      EGL、Dear ImGui context、GL 纹理、渲染快照
Lua 事件泵子线程     等待 C++ 事件，取得 VM Gate 后调用 Lua 回调
Lua 主/用户子线程    创建和更新模型；同一时刻由 VM Gate 串行访问 Lua VM
```

渲染线程永远不调用 Lua。事件等待期间释放 VM Gate，所以网络请求、`sleep` 或其他 Lua
子线程不会因为 ImGui 显示而停止调度。核心模型使用互斥锁和不可变快照，后续 JS、Go 与
插件可从其他线程调用同一 C ABI。

ImGui 事件泵是引擎内部子任务，仍参与统一停止和最终 `join`，但不占用脚本公开的 10 个
`beginThread` / `newThread` 用户子线程名额。

## 生命周期

- 同一脚本只有一个 ImGui Surface 和一个渲染线程；重复 `show` 会更新宿主。
- `close` 后允许在同一脚本再次 `show`，内部事件泵保持可复用。
- `show(true)` 在按钮或窗口回调调用 `close` 后返回 `true`。
- Surface、WindowManager 或 EGL 异步创建失败会唤醒 `show(true)`，返回 `false` 并保留错误。
- 脚本正常结束、错误、外部停止、`exitScript` 和强停进程都会关闭 Surface，停止渲染线程，
  并释放句柄、Lua 回调、图片点阵、GL 纹理和 Dear ImGui context。
- `showWindow` 本身不阻塞；脚本主任务结束后窗口会按上述规则自动关闭。
