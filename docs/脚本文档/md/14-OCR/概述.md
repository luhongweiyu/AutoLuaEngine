`m.ocr` 使用 RapidOCR 兼容的 PP-OCR ONNX 检测、识别和可选方向分类模型。APK 已内置
PP-OCRv4 mobile 中文/英文模型，可直接调用 `loadBuiltin()`，不联网、不额外下载；需要其他
语言或自训练模型时继续使用 `load(...)` 指定文件。模型不再使用时调用 `release(name)`。

内置模型流程：

```lua
capture("/sdcard/xiaoyv/scripts/current.png")
m.ocr.loadBuiltin() -- 默认模型名称为 builtin
local result = m.ocr.read("builtin", "/sdcard/xiaoyv/scripts/current.png")
m.ocr.release("builtin")
```

自定义模型文件必须是 Android 可直接读取的普通文件路径。建议与脚本放在共享存储目录，例如：

```text
/sdcard/xiaoyv/models/dbnet.onnx
/sdcard/xiaoyv/models/crnn.onnx
/sdcard/xiaoyv/models/angle.onnx
/sdcard/xiaoyv/models/keys.txt
```

自定义模型流程：

```lua
capture("/sdcard/xiaoyv/scripts/current.png")
m.ocr.load("main", detPath, recPath, clsPath, keysPath)
local result = m.ocr.read("main", "/sdcard/xiaoyv/scripts/current.png")
m.ocr.release("main")
```

同一模型名称、相同配置的重复 `load` 会直接复用已加载 session，不增加引用次数。不同名称使用
相同配置时也会共享底层 session，所有名称都 `release` 后才真正释放 ONNX 模型内存。
