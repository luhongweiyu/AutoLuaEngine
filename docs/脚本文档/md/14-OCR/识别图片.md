params: "name, imagePath: string, options: table?"
returns: "table 或 nil, string"

**方法名称：** 识别图片中的全部文字。

**语法：** `m.ocr.read(name, imagePath[, options])`

**参数说明：**

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `name` | `string` | 是 | 已加载的 OCR 模型名称。 |
| `imagePath` | `string` | 是 | 需要识别的 PNG、JPEG、WebP 等普通图片文件路径。 |
| `options` | `table?` | 否 | 识别配置。省略或传入空表时使用全部默认值。 |

**`options` 详细说明：**

| 字段 | 类型 | 默认值 | 有效范围 | 作用 |
|---|---|---:|---:|---|
| `threshold` | `number` | `0.30` | `0.01`～`0.99` | 文本检测概率图的像素二值化阈值。概率不低于该值的像素才会参与组成候选文本区域。调低可发现颜色较淡、较细或对比度较低的文字，但会增加噪点和错误区域；调高可减少背景干扰，但可能漏掉弱文字。 |
| `boxThreshold` | `number` | `0.50` | `0.01`～`0.99` | 候选文本区域的平均检测置信度阈值。区域平均分低于该值时，不会送入文字识别模型。调低可保留更多疑似文字区域；调高可减少误检，并减少后续需要识别的文本框数量。 |
| `unClipRatio` | `number` | `1.50` | `0.00`～`4.00` | 按候选框尺寸向外扩张文本框。适当扩张可以避免字符边缘被裁掉；过大会把相邻文字或背景一起截入，反而降低识别率。`0` 表示不做这一步的比例扩张。 |
| `maxSideLen` | `integer` | `960` | `32`～`2048` | 检测模型输入图像的最长边上限。仅对动态尺寸检测模型生效；原图最长边不超过该值时不会主动放大。值越大越有利于检测小字，但检测耗时和临时内存也会增加。固定输入尺寸的检测模型使用模型自身尺寸，此字段无效。 |
| `padding` | `integer` | `2` | 不小于 `0` | 在 `unClipRatio` 扩张完成后，再按原图像素向文本框四周增加固定边距。适合补回被裁掉的一两列笔画。数值过大会包含邻近文字；超出图片的部分会自动限制在图片边界内。 |
| `useAngle` | `boolean` | `false` | `true` / `false` | 是否调用方向分类模型判断文本框需要不需要旋转 `180` 度。它不负责纠正 `90` 度、`270` 度或任意倾斜角度。开启后每个文本框会多执行一次分类推理；自定义模型没有加载 `clsPath` 时，即使设为 `true` 也不会执行。内置模型包含方向分类模型。 |
| `minScore` | `number` | `0.00` | `0.00`～`1.00` | 最终文字识别置信度下限。识别结果的 `score` 低于该值时不会加入 `items`。该过滤发生在文字已经识别之后，因此不能减少推理耗时；调高可以去掉不可靠结果，但也可能过滤难识别的正确文字。 |
| `maxRecognizerWidth` | `integer` | `1024` | `32`～`4096` | 单个文本框送入动态宽度识别模型时允许使用的最大输入宽度。长文本行识别不完整时可适当调高；值越大，长文本框占用的推理时间和临时内存越多。固定宽度识别模型使用模型自身宽度，此字段无效。 |

传入范围外的数值会被限制到表中的有效范围；`padding` 的负数会按 `0` 处理。

检测阶段会依次执行：按 `maxSideLen` 缩放图片、使用 `threshold` 生成前景像素、使用
`boxThreshold` 筛选候选区域、再通过 `unClipRatio` 和 `padding` 扩张文本框。随后才会执行
可选的 `useAngle` 方向分类、文字识别和 `minScore` 结果过滤。因此：

- `threshold` 判断“哪些像素像文字”，`boxThreshold` 判断“整个候选区域是否可信”，两者不是同一个阈值。
- `unClipRatio` 是随文本框大小变化的比例扩张，`padding` 是原图坐标中的固定像素扩张。
- 返回项中的 `score` 是文字识别置信度，不是 `boxThreshold` 使用的检测区域分数。
- `minScore` 只过滤返回结果；需要提升速度时，应优先降低 `maxSideLen` 或关闭 `useAngle`。

常见配置参考：

```lua
-- 默认均衡配置：options 可以直接省略。
local result = m.ocr.read("builtin", imagePath)

-- 小字、浅色字或低对比度文字：提高检测分辨率并适当放宽检测阈值。
local result = m.ocr.read("builtin", imagePath, {
    maxSideLen = 1280,
    threshold = 0.25,
    boxThreshold = 0.40,
})

-- 背景复杂、误识别较多：提高候选框和最终识别结果的可信度要求。
local result = m.ocr.read("builtin", imagePath, {
    boxThreshold = 0.60,
    minScore = 0.60,
})

-- 优先速度：降低检测分辨率，并且不执行方向分类。
local result = m.ocr.read("builtin", imagePath, {
    maxSideLen = 640,
    useAngle = false,
})

-- 图片中可能存在上下颠倒的文字。
local result = m.ocr.read("builtin", imagePath, {
    useAngle = true,
})
```

建议一次只调整一到两个字段：漏检文字先调整 `maxSideLen`、`threshold` 和
`boxThreshold`；字符边缘不完整再调整 `unClipRatio` 或 `padding`；长文本行识别不完整再
调整 `maxRecognizerWidth`；只想隐藏低可信结果则调整 `minScore`。

| 返回值 | 说明 |
|---|---|
| `table` | 成功时返回 `{ items = { ... } }`。每项包含 `text:string`、`x/y/w/h:integer`、`score:number`；坐标和尺寸均对应原始图片，`score` 范围为 `0.00`～`1.00`。没有符合条件的文字时，`items` 是空表。 |
| `nil, string` | 模型未加载、图片无法解码或推理失败时返回错误信息。 |

**详细说明：**

```lua
local imagePath = "/sdcard/xiaoyv/scripts/current.png"
assert(capture(imagePath))

local result, errorMessage = m.ocr.read("main", imagePath, {
    useAngle = true,
    minScore = 0.5,
})
if not result then
    print("OCR 失败：", errorMessage)
    return
end

for _, item in ipairs(result.items) do
    print(item.text, item.x, item.y, item.w, item.h, item.score)
end
```

识别只接受图片路径，推荐先用 `capture` 保存当前缓存帧再调用。推理期间 Lua native 子线程
会让出 VM Gate；同一模型的 ONNX session 仍会按顺序执行，避免模型释放和推理并发冲突。
