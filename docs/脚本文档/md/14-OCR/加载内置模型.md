**方法名称：** 加载内置 OCR 模型。

**语法：** `m.ocr.loadBuiltin([name[, threads]])`

**参数说明：**

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `name` | `string?` | 否 | 脚本使用的模型名称，默认 `builtin`。 |
| `threads` | `integer?` | 否 | ONNX 推理线程数，默认 `2`，范围 `1` 到 `32`。 |

| 返回值 | 说明 |
|---|---|
| `boolean` | 成功加载或复用已有模型时返回 `true`。 |
| `nil, string` | 内置资源准备、ONNX session 创建或参数校验失败时返回错误信息。 |

**详细说明：**

```lua
assert(m.ocr.loadBuiltin())

local imagePath = "/sdcard/xiaoyv/scripts/current.png"
assert(capture(imagePath))

local result, errorMessage = m.ocr.read("builtin", imagePath, {
    useAngle = true,
})
if not result then
    print("OCR 失败：", errorMessage)
end

m.ocr.release("builtin")
```

APK 内置 RapidOCR PP-OCRv4 mobile 中文/英文检测、识别、方向分类模型和匹配字典，不需要联网
或额外下载。首次调用会把约 `15.5 MiB` 模型资源复制到应用私有目录并创建 ONNX session；
后续调用直接复用私有文件。设置 `useAngle = true` 时使用内置方向分类模型。

模型名称遵循与 `m.ocr.load()` 相同的管理规则：同名、相同线程数重复加载直接复用，不增加引用
次数；同名已加载其他配置时返回错误，需先 `release(name)`。`release` 释放 session 内存，不删除
应用私有模型文件；卸载 App 时由系统一并清除。
