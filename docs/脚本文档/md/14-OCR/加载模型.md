**方法名称：** 加载 RapidOCR ONNX 模型。

**语法：** `m.ocr.load(name, detPath, recPath, clsPath, keysPath[, threads])`

**参数说明：**

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `name` | `string` | 是 | 当前脚本使用的模型名称。 |
| `detPath` | `string` | 是 | 文本检测模型 `.onnx` 路径。 |
| `recPath` | `string` | 是 | 文本识别模型 `.onnx` 路径。 |
| `clsPath` | `string?` | 否 | 方向分类模型 `.onnx` 路径；传 `nil` 时不加载方向分类。 |
| `keysPath` | `string` | 是 | 与识别模型匹配的字符字典文本路径。 |
| `threads` | `integer?` | 否 | ONNX 推理线程数，默认 `2`，范围 `1` 到 `32`。 |

| 返回值 | 说明 |
|---|---|
| `boolean` | 成功加载或复用已有模型时返回 `true`。 |
| `nil, string` | 路径、模型结构或字典不匹配时返回错误信息。 |

**详细说明：**

```lua
local modelDir = "/sdcard/xiaoyv/models"
local ok, errorMessage = m.ocr.load(
    "main",
    modelDir .. "/dbnet.onnx",
    modelDir .. "/crnn_lite_lstm.onnx",
    modelDir .. "/angle_net.onnx",
    modelDir .. "/keys.txt",
    2
)

if not ok then
    print("OCR 模型加载失败：", errorMessage)
end
```

首次加载会创建 ONNX session，可能耗时较长。Lua native 子线程调用此方法时会暂时让出 VM Gate，
不阻塞其他可运行的 Lua 任务。
