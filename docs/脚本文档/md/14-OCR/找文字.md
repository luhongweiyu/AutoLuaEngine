returns: "table 或 nil, string"

**方法名称：** 在 OCR 结果中查找文字。

**语法：** `m.ocr.findText(name, imagePath, text[, options])`

**参数说明：**

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `name` | `string` | 是 | 已加载的 OCR 模型名称。 |
| `imagePath` | `string` | 是 | 图片文件路径。 |
| `text` | `string` | 是 | 要查找的文字。默认按包含关系匹配。 |
| `options` | `table?` | 否 | 支持 `m.ocr.read` 的全部识别配置；各字段含义、默认值和范围见[识别图片](识别图片.md)。额外支持 `exact:boolean`，默认 `false`。为 `false` 时按包含关系匹配，为 `true` 时要求识别结果与 `text` 完全相等。该字段只改变文字匹配方式，不改变 OCR 识别过程。 |

| 返回值 | 说明 |
|---|---|
| `table` | 成功时返回 `{ found = boolean }`；找到时还包含 `x, y, w, h, text, score`。 |
| `nil, string` | 模型、图片或推理失败时返回错误信息。 |

**详细说明：**

```lua
local result, errorMessage = m.ocr.findText(
    "main",
    "/sdcard/xiaoyv/scripts/current.png",
    "设置",
    { exact = true, useAngle = true }
)

if not result then
    print("OCR 找字失败：", errorMessage)
elseif result.found then
    print("找到：", result.x, result.y, result.w, result.h)
end
```

未找到属于正常结果，返回 `result.found == false`，不会返回错误。
