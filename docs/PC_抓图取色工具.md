# PC 抓图取色工具

## 定位

`xiaoyv_tools.exe` 是小鱼精灵的独立桌面图像工具，使用 C++20 和 Qt 6 Widgets 开发。
它既可独立运行，也可由 VSCode 插件启动。两个工具分别连接 Android 引擎，互不承担中转：

```text
VSCode 插件 ----独立 ADB forward / LAN----\
                                        Android EngineHttpServer
抓图取色器 ----独立 ADB forward / LAN----/
```

最终交互规则以 [PC 抓图取色器行为契约](PC_抓图取色器_行为契约.md) 为准。

## 源码边界

```text
ide/xiaoyv-tools/
  formats/                 内置及用户可编辑的 Lua/JavaScript 代码生成格式
  src/model/               图片、取色点、点阵字库状态和不变量
  src/core/                QImage 与跨平台 image_core 的适配
  src/workspace/           图片标签、保存关闭、待用框选和当前画布
  src/canvas/              图片标签、画布、放大镜、键盘定位和框选
  src/panels/              取色、二值化、点阵提取、字库编辑和脚本编辑
  src/device/device_transport.*  LAN/ADB、HTTP 和 JSON-RPC
  src/device/device_client.*     截图、投影、脚本和日志业务
  src/capture/             浮动区域抓图、Windows 指定窗口抓图
  src/generator/           格式发现及受限 Lua/JavaScript 执行
  src/ui/                  动作目录、主题、图标和停靠窗
  src/dialogs/             连接设置等对话框
  tests/                   模型、界面、设备协议和主窗口测试

shared/native/image_core/
  image_core.h/.cpp        Android 与 PC 共用的颜色和点阵算法
```

主窗口只负责装配菜单、停靠窗、状态栏和跨模块信号，不保存第二份图片、取色、字库或设备状态。菜单和主工具栏的动作定义
集中在 `src/ui/action_catalog.h`，文字、图标、顺序和分组不再分别维护。

`ImageWorkspace` 是图片标签和待用范围的唯一所有者；`FontDictionaryEditor` 是字库文件与冲突事务的唯一所有者；
`DeviceTransport` 不包含截图或脚本状态，`DeviceClient` 不直接接触 ADB 进程和网络响应。

## 图片与标签

- 支持 Qt 可读取的 PNG、JPEG、BMP、WebP、GIF、TIFF、ICO 等图片，也支持拖入图片文件或
  图片数据。
- 磁盘图片属于文件图片；设备截图、浮动抓图、窗口截图、拖入图片和裁剪结果属于临时图片。
- 图片统一转为 RGBA8888 后进入文档；状态栏只显示坐标、`RRGGBB`、缩放和分辨率，不重复显示通道格式或十进制 RGB。
- 临时图片和已修改文件图片在标签后显示 `*`。关闭单个临时标签直接丢弃，不弹保存提示；
  关闭修改过的文件图片时才询问保存。
- 退出程序时先处理未保存字库，再汇总显示临时图片和修改文件图片数量，只询问一次。
- 保存时未填写扩展名会自动补 `.png`。
- 撤销历史最多保留 24 个图片状态，并受 256 MiB 估算点阵预算限制。

## 画布交互

- 没有图片时仍显示中性灰网格画布，面板参数仍可输入。
- 新图片默认 `1:1` 从左上角显示，图片不能在画布中自由拖动；超出区域使用滚动条。
- 鼠标滚轮保持标准滚动语义，缩放只由工具栏按钮触发。
- 支持左右旋转、水平翻转、上下翻转、适应窗口、实际大小、撤销和重做。
- 旋转和翻转会生成真实的新点阵和宽高；框选范围随图片几何变换，已有取色点不被静默改写。
- 高倍率像素网格只绘制当前可见区域，不遍历整张大图。
- 鼠标指向图片时显示跟随式 `23 x 23` 像素放大镜，中心像素用红框标识。
- 普通单击只定位，不取色。回车或空格采集当前像素；方向键每次移动一个图片像素，并同步
  逻辑坐标和系统鼠标。程序移动鼠标产生的 DPI/缩放舍入事件不会反向覆盖目标坐标。
- 框选时鼠标可移出图片区域，坐标持续更新并限制到图片边界；释放后退出框选模式，临时框
  立即消失。
- 点阵字库标签激活时，以黑白双色虚线显示已确认范围；离开该标签后隐藏。

## 取色与生成代码

- 取色点数量不限，并在图片标签之间连续沿用；切换图片不会换成另一份列表或清空。
- 表格包含启用、序号、颜色块、坐标、RGB、偏色、唯一基准点和逐行删除按钮；坐标、RGB 与偏色支持双击编辑，单击颜色块打开颜色选择器。整行选中背景不会遮住颜色块。
- 基准点被禁用时，生成代码临时使用第一个启用点；不会擅自修改用户保存的基准状态。
- 刷新选中或刷新全部时读取当前可见图片；旋转、翻转或二值化不会自动刷新历史取色点。
- 取色与点阵字库共享当前会话的框选范围，切换图片时直接沿用，两个面板不保存范围副本。手动范围只检查语法和坐标顺序，不依据图片尺寸
  静默修改；真正处理图片时才与边界求交。
- 没有图片时仍保留当前范围，下一张图片继续采用。
- 每个生成格式只对应一种输出语言，选择格式后自动确定语言；默认优先选择 Lua
  `m.findColors`。

格式目录：

```text
formats/<格式目录>/
  format.json
  generator.lua            或 generator.js，只允许一个生成器
```

`format.json`：

```json
{
  "id": "m-find-colors-lua",
  "name": "小鱼 m.findColors (Lua)",
  "language": "lua",
  "generator": "generator.lua"
}
```

格式统一从 exe 同级的 `formats` 目录读取。内置格式随程序放在这里，用户可以直接修改、删除
或新增格式；点击“打开格式目录”即可打开该目录。重复 ID、无效 JSON、缺失文件和不支持的扩展名
会显示为加载错误，不会静默消失。Lua 生成器有指令和
两秒时间预算，JavaScript 生成器有两秒中断边界，单个生成器文件最大 1 MiB。

生成上下文保持统一：

```text
points[]    enabled/base/x/y/dx/dy/hex/delta
region      left/top/right/bottom
image       width/height/path
direction   1..8
defaultDelta
```

## 二值化与偏色

- 颜色模式使用与点阵字库相同的 `RRGGBB-偏色|...` 规则输入框；启用的取色点变化后自动同步，也可手动修改。规则数量不限制为单个颜色。
- 灰度模式使用 `0..255` 阈值；颜色规则与灰度模式互斥，未启用参数明显禁用，切换时布局不抖动。
- 只有“二值化 / 偏色”标签可见时显示预览；参数变化自动刷新，离开标签或隐藏图像工具时立即恢复原图显示，取色时不会把图片变成黑白。
- 支持反色和应用，不提供预览开关、预览按钮或还原按钮。
- 预览在工作线程计算，主线程同一时间只接收一个任务，连续修改只保留最后请求。
- 结果必须同时匹配当前文档、图片版本、参数和请求号，否则直接丢弃。
- “应用”没有可用预览时会计算最新参数并在完成后应用，不会使用上一轮结果。
- 只有“应用”修改原图并进入撤销历史；预览不会改变保存状态。

## 点阵字库

- 颜色规则格式为 `RRGGBB-偏色`，多条用 `|` 分隔。取色点变化时会同步到颜色框，也可
  手动修改。
- 提取结果第一行是未分割点阵，后续结果按空白行和空白列间隔分割；即使只有一个字形且与第一行点阵相同，也保留该分割结果。
- 结果表格依次显示“序、文字、宽高、点数”。有分割结果时默认选中第一个分割字形，第一行由用户主动选择后再显示点阵网格。
- 二值化参数、点阵提取结果、文字、颜色规则和当前选择属于工具会话；切换图片后保持上一张的状态，不按图片分别保存。
- 点击“提取点阵”时只清空当前图片的旧结果；规则无效或没有前景点时不保留该图片上一轮预览。
- 每条结果都可填写文字。批量加入只处理已填写文字的行；取消批量冲突决策时，临时事务
  整体丢弃，不留下半批修改。
- 点阵文本为 `文字$宽$高$十六进制`；没有文字时为 `宽$高$十六进制`。
- 点阵预览使用固定 `6 x 6` 格子，宽高再大也不缩放；超出区域使用滚动条。
- 网格默认只读。勾选“允许编辑”后，单击格子才会反转前景点并同步点阵文本。
- 同一文字允许多个不同点阵；同一点阵只允许对应一个文字；相同文字和相同点阵直接跳过。
- 同一点阵遇到不同文字时，现场选择跳过、替换或取消；批量时可选择全部跳过或全部替换。
- 字库列表直接使用 `FontDictionaryDocument` 作为 Qt 数据模型，新增、替换、删除和磁盘合并只有一个记录源。
- “替换选中”也经过同一点阵唯一性检查，不能绕过规则制造重复。
- 字库打开后保留未识别行；含 `$` 但无法解析的行会提示，并在保存时原样保留。
- 保存前重新读取磁盘，保留外部新增记录和注释。本地修改与外部修改冲突时现场决策；使用
  `QSaveFile` 原子提交，失败时原文件不变。
- “当前图片测试”位于“区域范围”同行，使用当前选中点阵终检并显示命中数量和首个坐标。

## 本机抓图

- 浮动抓图显示透明置顶选区，可拖动内部移动，可拖动边缘或四角缩放。点击“抓图”时临时
  隐藏选区，仅抓取框内屏幕，完成后恢复原选区，方便连续抓图。
- Windows 窗口截图按钮单击时抓取已指定窗口；按住拖动时靶心变红，状态栏实时显示目标
  标题、类名和尺寸，释放后记录目标。未指定窗口时单击会提示拖动靶心。
- 指定窗口使用 `PrintWindow(PW_RENDERFULLCONTENT)`，不会把遮挡在目标上方的其他窗口
  混入截图；目标明确不支持时直接报告失败。

## 设备连接

- 默认局域网直连，也可使用 ADB。ADB 模式通过 `adb forward tcp:0` 申请本工具独立动态
  端口，不影响 VSCode 插件的连接。
- ADB 转发使用异步 `QProcess`，多个同时到达的请求共用一次端点建立；ADB 无法启动时
  所有等待操作都会收到明确失败。
- 启动和修改设置后自动检查一次连接，状态栏显示“正在连接/已连接/未连接”；手动刷新只保留在设备菜单。
- 设备操作直接执行，不在每条命令前额外检查；收到 HTTP 响应或发生传输失败时自动更新状态，HTTP/JSON-RPC 业务错误不会误报断线。设备截图和投影分别有忙状态，重复点击不会重复提交。
- 设备截图读取 `XYVF + width + height + RGBA8888` 原始帧，严格检查宽高和点阵长度。
- 投影先在工作线程编码 PNG，再执行固定两步协议：

```text
PUT /tool/image
viewer.openImage(fileId)
```

- 脚本运行前读取日志位置，运行中每 500 ms 拉取新增日志，并提供停止按钮。普通请求有
  超时，`script.run` 不使用普通短超时；脚本完成后最后读取一次日志再结束状态。
- 网络失败会使当前 ADB 动态端点失效，下一次操作重新建立，不执行多套业务命令兜底。
- 已创建的动态端口不会因失效而丢失记录，切换连接设置或退出程序时会逐个清理，不影响 VSCode 插件的转发。
- 停止脚本请求返回后，停止按钮保持禁用，直到原脚本运行请求真正结束。

## 界面与主题

- 默认中性暗色，另有清爽亮色、深夜蓝灰、碳黑暖灰、柔和浅色和高对比主题。
- 亮暗主题使用不同前景色重新绘制高 DPI 工具图标，不缩放低分辨率位图。
- 主窗口几何、停靠布局、面板显隐和主题由 `QSettings` 保存。
- 图像工具、生成代码和测试日志使用可停靠窗口。嵌入时不显示普通标题栏，只在右上角
  保留小操作柄；点击柄可浮动或关闭。悬浮后使用匹配主题的自定义标题栏，并保留 Qt 原生
  拖动停靠和预定位阴影。
- 图像工具使用紧凑的“取色、二值化、字库”标签，右侧停靠列默认最窄可缩至约 `280px`。
- 生成代码位于图像工具下方，复制、运行和停止按钮左对齐。
- 测试日志默认只占约三到五行且只位于中央画布下方，右下角归右侧停靠列，不挤压图像工具和生成代码。
- 停靠区分隔条固定为 `3px`，生成代码内容贴合停靠边界，不保留额外的上下左右占位。
- 悬浮标题栏只在浮动状态显示；重新嵌入或程序启动恢复布局时必须显式隐藏，不能残留“生成代码 ×”覆盖按钮行。

## 构建与测试

环境：Visual Studio 2022 C++、CMake、Ninja、Qt 6.8。

```powershell
cd ide/xiaoyv-tools
.\构建.ps1 -Configuration Release
```

构建脚本依次执行配置、编译、CTest 和 `windeployqt`。测试覆盖：

- 图片临时/文件状态、撤销、预览版本和二值化。
- 点阵提取、同字多点阵、点阵唯一性、外部文件合并和冲突决策。
- 画布方向键、范围边界和异步预览失效。
- LAN 健康检查、原始截图帧和重复请求防抖。
- 脚本停止请求防重复，以及停止后运行会话的最终收束。
- 动作目录、主题、主窗口和连接设置构造。

输出：`ide/xiaoyv-tools/build/xiaoyv_tools.exe`。

## 跨平台边界

- `shared/native/image_core` 不依赖 Qt 或 Android，可继续供 Windows、Android、iOS 和插件复用。
- 图片模型、点阵字库模型和生成器不依赖具体设备平台。
- `DeviceTransport` 封装统一 HTTP/JSON-RPC 协议，`DeviceClient` 只实现设备业务；新增设备平台不修改图像工作流。
- 浮动桌面区域抓图使用 Qt；指定窗口抓图是 Windows 平台实现，其他平台后续增加对应捕获层。
