# AutoLuaEngine 脚本文档

当前 Android + Lua 脚本层包含固定 HostApi 和动态 Java 互操作。固定自动化函数通过
`system_c_api` 调用 `libengine.so/core/api`；`import` 由 `libengine.so` 的 Lua
运行时桥接 JNI，不为每个 Java 方法重复生成固定 API。

## 已可用 API

| 分类 | 函数 | 说明 |
|---|---|---|
| 全局 | `print(...)` | 输出到引擎日志 |
| 全局 | `sleep(ms)` | 脚本延时，支持停止请求 |
| 全局 | `systemTime()` | 返回系统时间戳，单位毫秒 |
| 全局 | `tickCount()` | 返回当前脚本运行时间，单位毫秒 |
| 全局 | `getRunEnvType()` | 返回当前运行环境类型 |
| 全局 | `useApi(name)` / `switchApi(name)` | 将 `m` / `lr` / `cd` 的一层函数导出到 `_G` |
| Java | `import(className)` | 导入 Java 类或包，兼容懒人精灵 |
| Java | `LuaEngine.getContext()` | 返回 Android Application Context |
| Java | `LuaEngine.httpGet/httpPost/httpPostData` | Java 层 HTTP 请求，兼容懒人精灵 |
| Java | `LuaEngine.loadApk(nameOrPath)` | 加载 APK/JAR/DEX 插件 |
| 延时 | `m.sleep(ms)` | 脚本延时，支持停止请求 |
| 时间 | `m.systemTime()` | 返回系统时间戳，单位毫秒 |
| 时间 | `m.tickCount()` | 返回当前脚本运行时间，单位毫秒 |
| 日志 | `m.log.print(text)` | 输出日志文本 |
| 输入 | `touchDown(id, x, y)` | 按住不放，仅 Root 模式 |
| 输入 | `touchMove(id, x, y)` | 移动手指，仅 Root 模式 |
| 输入 | `touchUp(id)` | 弹起手指，仅 Root 模式 |
| 输入 | `keyDown(keycode)` | 按下按键不弹起，仅 Root 模式 |
| 输入 | `keyUp(keycode)` | 弹起按键，仅 Root 模式 |
| 输入 | `keyPress(keycode)` | 按一下按键并弹起，仅 Root 模式 |
| 输入 | `inputText(text)` | 模拟输入文字，仅 Root 模式 |
| 截图 | `m.capture()` | 返回 `width, height, pixelsAddress` |
| 截图 | `m.keepCapture()` | 锁住当前截图帧 |
| 截图 | `m.releaseCapture()` | 取消锁帧，恢复按时间缓存 |
| 截图 | `m.setCaptureCacheMs(ms)` | 设置截图缓存时间 |
| 找色 | `m.findColors(x1, y1, x2, y2, dir, sim, colors)` | 在当前截图缓存上多点找色 |

`lr` / `cd` 当前映射同一组最小函数：`print`、`sleep`、`systemTime`、`tickCount`、`getRunEnvType`、`touchDown`、`touchMove`、`touchUp`、`keyDown`、`keyUp`、`keyPress`、`inputText`、`capture`、`keepCapture`、`releaseCapture`、`setCaptureCacheMs`、`findColors`。

## Java 互操作

`import` 的 Lua 写法、Java 对象成员访问和返回值行为兼容懒人精灵。

```lua
import("java.lang.*")
import("java.util.*")
import("android.content.Context")
import("com.nx.assist.lua.LuaEngine")

print(Math.sin(1.2))
print(System.currentTimeMillis())

local context = LuaEngine.getContext()
print(context.getPackageName())
print(Context.ACTIVITY_SERVICE)

local values = ArrayList()
values.add("b")
values.add("a")
print(values.size(), values[0], #values)
```

支持静态/实例字段、静态/实例方法、构造函数、内部类、重载、链式调用、Java
数组与集合、Lua table 转数组/集合/Map，以及 Lua function/table 转 Java 接口。

```lua
local runnable = Runnable(function()
    print("Java Runnable 回调")
end)
runnable.run()
```

Java 数组和 `List` 在 Lua 中使用 Java 的 `0` 起始下标。Java 异常会转换为 Lua
错误，可以通过 `pcall` 捕获。完整实现规则见 `docs/ANDROID_Java互操作.md`。

## 时间

```lua
local tm = systemTime()
print(tm)

local tick = tickCount()
print(tick)
```

说明：

- `systemTime()` 返回 Unix epoch 毫秒时间戳。
- `tickCount()` 返回当前脚本从开始执行到现在的毫秒数。
- 两个函数也可以通过 `m.systemTime()`、`m.tickCount()` 调用。

## 输入注入

输入注入当前只走 Root helper 常驻进程，不走无障碍，也不为每个命令拉起外部
`input` 进程。Root 不可用时返回 `false`，`touchDown` / `touchUp` 按兼容语义不返回值。

### 触摸

```lua
touchDown(0, 100, 100)
sleep(50)
local ok = touchMove(0, 200, 200)
touchUp(0)
print(ok)
```

参数：

- `id`：模拟手指索引，范围 `0` 到 `4`。
- `x, y`：屏幕坐标，整数。

返回：

- `touchDown`：无返回值。
- `touchMove`：返回布尔值。
- `touchUp`：无返回值。

### 按键

```lua
keyDown("home")
sleep(100)
keyUp("home")

local ok = keyPress("Back")
print(ok)
```

支持的按键标识符：

| 名称 | 标识符 | 按键码 |
|---|---|---|
| 主页键 | `Home` | `3` |
| 返回键 | `Back` | `4` |
| 打电话 | `Call` | `5` |
| 挂电话 | `EndCall` | `6` |
| 音量增加 | `VolUp` | `24` |
| 音量减少 | `VolDown` | `25` |
| 电源键 | `Power` | `26` |
| 相机键 | `Camera` | `27` |
| 菜单键 | `Menu` | `82` |
| 向上翻页 | `PageUp` | `92` |
| 向下翻页 | `PageDown` | `93` |

`keycode` 可以传标识符，也可以直接传数字按键码。

### 文本

```lua
local ok = inputText("hello nice!!!")
print(ok)
```

`inputText` 当前通过 Root 注入按键事件实现，适合英文、数字和常见符号。中文输入、
组合输入和候选词输入后续更适合走独立输入法能力。

### 运行环境

```lua
print(getRunEnvType())
```

当前返回：

- `root`：Root helper 已就绪。
- `none`：当前没有可用 Root 输入注入环境。

## 截图返回值

```lua
local w, h, pixels = m.capture()
if not w then
    print("截图失败：", h)
    return
end

print("宽 =", w)
print("高 =", h)
print("点阵地址 =", string.format("0x%x", pixels))
```

成功时：

- 第 1 个返回值：宽度。
- 第 2 个返回值：高度。
- 第 3 个返回值：点阵地址。
- 点阵格式固定为紧凑 RGBA，长度为 `w * h * 4`。

失败时：

- 第 1 个返回值：`nil`。
- 第 2 个返回值：错误信息。

## 缓存规则

- 默认缓存时间：`20ms`。
- 缓存时间内重复调用 `m.capture()`，复用当前点阵。
- 超过缓存时间后重新截图并覆盖内部缓存。
- `m.keepCapture()` 后一直复用当前帧。
- `m.releaseCapture()` 后恢复按时间缓存。
- 脚本开始和结束会清空截图缓存。

## 找色

```lua
local x, y = m.findColors(0, 0, 1079, 1919, 2, 0x101010, "0|0|FFFFFF,10|5|FF0000-101010")
if not x then
    print("找色失败：", y)
    return
end

print("找到：", x, y)
```

参数：

- `x1, y1, x2, y2`：查找范围。
- `dir`：扫描方向，取值 `1` 到 `8`，沿用旧找色算法。
- `sim`：默认容差，格式为 `0xRRGGBB`。
- `colors`：多点颜色字符串，格式为 `x|y|RRGGBB`，可用 `-RRGGBB` 给单点设置独立容差。

返回：

- 找到：返回 `x, y`。
- 未找到或失败：返回 `nil, errorMessage`。

说明：

- 找色直接使用当前截图缓存，不需要“是否截屏”参数。
- 如果要固定同一帧连续找色，先调用 `m.keepCapture()`；完成后调用 `m.releaseCapture()`。

## C ABI 对应关系

Lua 当前函数对应的 C ABI：

```text
print(...)                -> engine_print
m.log.print(text)         -> engine_logPrint
m.sleep(ms)               -> engine_sleepInterruptible
systemTime()              -> engine_systemTime
tickCount()               -> engine_tickCount
touchDown(...)            -> engine_touchDown
touchMove(...)            -> engine_touchMove
touchUp(...)              -> engine_touchUp
keyDown(keycode)          -> engine_keyDown
keyUp(keycode)            -> engine_keyUp
keyPress(keycode)         -> engine_keyPress
inputText(text)           -> engine_inputText
getRunEnvType()           -> engine_getRunEnvType
m.capture()               -> engine_capture
m.keepCapture()           -> engine_keepCapture
m.releaseCapture()        -> engine_releaseCapture
m.setCaptureCacheMs(ms)   -> engine_setCaptureCacheMs
m.findColors(...)         -> engine_findColors
```

`import` 是动态 Java 对象桥，不对应单个固定 C ABI。它的公开入口和 Lua userdata
元方法仍由 `libengine.so` 注册，Java 层只执行 JVM 类加载与反射。

相关 C ABI 声明：

```c
int engine_print(const char* text);
int engine_logPrint(const char* text);
int engine_sleepInterruptible(
        int durationMs,
        runtime_interrupt_callback shouldInterrupt,
        void* userData
);
long long engine_systemTime();
long long engine_tickCount();
int engine_touchDown(int id, int x, int y);
int engine_touchMove(int id, int x, int y);
int engine_touchUp(int id);
int engine_keyDown(const char* keyCode);
int engine_keyUp(const char* keyCode);
int engine_keyPress(const char* keyCode);
int engine_inputText(const char* text);
const char* engine_getRunEnvType();
const char* engine_inputLastError();
int engine_capture(int* width, int* height, unsigned char** pixels);
void engine_keepCapture();
void engine_releaseCapture();
int engine_setCaptureCacheMs(int durationMs);
void engine_clearCaptureCache();
const char* engine_captureLastError();
int engine_findColors(
        int x1,
        int y1,
        int x2,
        int y2,
        int dir,
        int sim,
        const char* colors,
        EnginePoint* point
);
const char* engine_findColorsLastError();
```
