# AutoLuaEngine 脚本文档

当前 Android + Lua 脚本层只保留已经按新 C ABI 整理完成的最小能力。

## 已可用 API

| 分类 | 函数 | 说明 |
|---|---|---|
| 全局 | `print(...)` | 输出到引擎日志 |
| 全局 | `useApi(name)` / `switchApi(name)` | 将 `m` / `lr` / `cd` 的一层函数导出到 `_G` |
| 延时 | `m.sleep(ms)` | 脚本延时，支持停止请求 |
| 日志 | `m.log.print(text)` | 输出日志文本 |
| 截图 | `m.capture()` | 返回 `width, height, pixelsAddress` |
| 截图 | `m.keepCapture()` | 锁住当前截图帧 |
| 截图 | `m.releaseCapture()` | 取消锁帧，恢复按时间缓存 |
| 截图 | `m.setCaptureCacheMs(ms)` | 设置截图缓存时间 |

`lr` / `cd` 当前只映射同一组最小函数：`print`、`sleep`、`capture`、`keepCapture`、`releaseCapture`、`setCaptureCacheMs`。

## 截图返回值

```lua
local w, h, pixels = m.capture()
if not w then
    print("截图失败：", h)
    return
end

print("宽 =", w)
print("高 =", h)
print("点阵地址 =", string.format("0x%x", pixels))
```

成功时：

- 第 1 个返回值：宽度。
- 第 2 个返回值：高度。
- 第 3 个返回值：点阵地址。
- 点阵格式固定为紧凑 RGBA，长度为 `w * h * 4`。

失败时：

- 第 1 个返回值：`nil`。
- 第 2 个返回值：错误信息。

## 缓存规则

- 默认缓存时间：`20ms`。
- 缓存时间内重复调用 `m.capture()`，复用当前点阵。
- 超过缓存时间后重新截图并覆盖内部缓存。
- `m.keepCapture()` 后一直复用当前帧。
- `m.releaseCapture()` 后恢复按时间缓存。
- 脚本开始和结束会清空截图缓存。

## C ABI 对应关系

Lua 截图函数最终调用 `libengine.so` 里的：

```c
int screen_capture(int* width, int* height, unsigned char** pixels);
void screen_keep_capture();
void screen_release_capture();
int screen_set_capture_cache_ms(int durationMs);
void screen_clear_capture_cache();
const char* screen_last_error();
```
