# 小鱼精灵 脚本文档

当前 Android + Lua 脚本层包含固定 HostApi、真实 native 多线程和动态 Java 互操作。固定自动化函数通过
`system_c_api` 调用 `libengine.so/core/api`；`import` 由 `libengine.so` 的 Lua
运行时桥接 JNI，不为每个 Java 方法重复生成固定 API。

## 已可用 API

Lua 仍是动态语言，下面的类型是 API 契约：`integer` 表示整数，`number` 表示整数或小数，
`string` 可承载文本或二进制数据，`table` 表示 Lua 表，`userdata` 表示 native/Java 对象，
`any` 表示任意 Lua 值，类型后的 `?` 表示该值可能为 `nil`。

`lr` / `cd` 当前保留已有的最小兼容映射。后续新增功能默认只进入 `m`，`lr` 和 `cd`
兼容层单独集中完善，不随每个新功能同步增加。

## Lua 多线程

```lua
local shared = { count = 0, running = true }

local thread = m.thread.newThread(function(step)
    while shared.running do
        shared.count = shared.count + step
        sleep(20)
    end
end, 2)

sleep(200)
shared.running = false
thread:stopThread()
print(shared.count)
```

每个子任务由真实 native 线程驱动，并使用 `lua_newthread` 共享 `_G`。同一时刻只有一个
任务执行 Lua 字节码；`sleep`、UI 等待和 Java 调用会让出 VM Gate。纯 Lua 计算存在
其他等待任务时，也会通过指令 hook 公平让出。

`callback` 类型必须为 `function`，后续 `...` 可以是任意 Lua 值并原样传给回调。
`m.thread.beginThread` 无返回值；`m.thread.newThread` 返回 `userdata` 线程对象；
`thread:stopThread()` 无参数、无返回值。

同时运行的子线程最多 10 个。主脚本结束时，引擎会停止并等待全部子线程退出。
`tickCount()` 在所有子线程中使用同一个顶层脚本开始时间。完整实现规则见
[Lua 多线程](LUA_多线程.md)。

## 加密脚本包

`.alpkg` 是可直接放进 Android 脚本目录运行的单文件包。项目根目录使用 `alpkg.json`：

```json
{
  "entry": "main.lua",
  "exclude": ["dist/", "*.bak"]
}
```

首次打包缺少 `alpkg.json` 时，打包器会自动创建上述默认配置并继续执行。

包内 Lua 文件会编译成无调试信息的 Lua 5.4.8 字节码并加密，原始 `.lua` 不写入包中；
`require("modules.task")`、`dofile("common/config.lua")`、`m.web.open({ file = "ui/index.html" })`
仍按项目相对路径使用。HTML、JS、CSS、图片等资源不加密，直接从包内读取。
HTML 内的 `href`、`src` 等相对路径同样以页面所在目录为基准，例如
`ui/index.html` 可以直接引用 `style.css`、`app.js` 和 `images/logo.png`。

### 读取包内资源

```lua
local css, errorMessage = m.read_alpkg_file("ui/style.css")
if not css then
    error(errorMessage)
end

local configText = assert(m.read_alpkg_file("data/config.json"))
local imageBytes = assert(m.read_alpkg_file("images/logo.png"))
```

`m.read_alpkg_file(path:string)` 返回二进制安全的 Lua `string`，适用于 CSS、JSON、HTML、图片和
其他非 Lua 文件。它只接受当前 `.alpkg` 的项目相对路径，只读取 `manifest.json` 中登记为
`resource` 的条目；不会解压到缓存目录、不会回退到包外文件，也不能读取加密 Lua 字节码。
读取失败返回 `nil, errorMessage:string`；普通 `.lua` 脚本调用时也会返回失败。

打包与格式说明见 [ALPKG 格式](../shared/package/ALPKG_格式.md)。

## Java 互操作

`import` 的 Lua 写法、Java 对象成员访问和返回值行为兼容懒人精灵。

```lua
import("java.lang.*")
import("java.util.*")
import("android.content.Context")
import("com.nx.assist.lua.LuaEngine")

print(Math.sin(1.2))
print(System.currentTimeMillis())

local context = LuaEngine.getContext()
print(context.getPackageName())
print(Context.ACTIVITY_SERVICE)

local values = ArrayList()
values.add("b")
values.add("a")
print(values.size(), values[0], #values)
```

支持静态/实例字段、静态/实例方法、构造函数、内部类、重载、链式调用、Java
数组与集合、Lua table 转数组/集合/Map，以及 Lua function/table 转 Java 接口。

```lua
local runnable = Runnable(function()
    print("Java Runnable 回调")
end)
runnable.run()
```

Java 数组和 `List` 在 Lua 中使用 Java 的 `0` 起始下标。Java 异常会转换为 Lua
错误，可以通过 `pcall` 捕获。完整实现规则见 `docs/ANDROID_Java互操作.md`。

## 脚本 UI

脚本 UI 分为三种：原生弹窗/表单、屏幕 HUD 和 HTML 页面。它们都在 App 主进程绘制，
Lua 脚本始终在 `:engine` 进程接收事件；脚本停止或执行结束时，当前脚本创建的 UI 会
被自动关闭。

### 原生弹窗和表单

`m.dialog.*` 和 `m.ui.form` 使用 WindowManager 悬浮窗口：不会压暗屏幕，不启动或带回
小鱼精灵 页面。点击框外区域会直接穿透到下方应用且不会关闭对话框；只能点击界面自身的
确认/取消按钮，或由脚本停止、引擎关闭时自动关闭。

原生弹窗、表单和 HUD 都需要 App 已取得悬浮窗权限；权限未开启时创建函数返回
`nil, errorMessage:string`。

```lua
local accepted, errorMessage = m.dialog.confirm("删除文件", "确定继续吗？")
if accepted == nil then
    print("弹窗失败：", errorMessage)
elseif accepted then
    print("用户确认")
end

local name, inputError = m.dialog.input("名称", "请输入名称", "默认值")

local index, value = m.dialog.select("运行模式", { "Root", "无障碍" }, 1)

local values = m.ui.form({
    title = "脚本参数",
    message = "填写后开始执行",
    fields = {
        { name = "account", label = "账号", type = "text", hint = "请输入账号" },
        { name = "password", label = "密码", type = "password" },
        { name = "retry", label = "重试次数", type = "number", default = "3" },
        { name = "enabled", label = "启用功能", type = "boolean", default = true },
        { name = "mode", label = "模式", type = "select", items = { "快", "稳" } },
    },
})
```

可用方法：

- `m.dialog.alert(title, message[, buttonText])`：三个参数均为 `string`，`buttonText` 可省略。
  确认或关闭返回 `boolean`；创建失败返回 `nil, errorMessage:string`。
- `m.dialog.confirm(title, message[, positiveText[, negativeText]])`：四个参数均为 `string`，
  后两个可省略。确认返回 `true`，取消或关闭返回 `false`，创建失败返回
  `nil, errorMessage:string`。
- `m.dialog.input(title, hint[, defaultText[, options]])`：前三个文本参数为 `string`，`options`
  为 `table`。确认返回 `text:string`；取消返回 `nil`；创建失败返回
  `nil, errorMessage:string`。
- `m.dialog.select(title, items[, selectedIndex[, options]])`：`title:string`、
  `items:table<string>`、`selectedIndex:integer`、`options:table`。确认返回
  `index:integer, value:string`，索引从 `1` 开始；取消返回 `nil`；创建失败返回
  `nil, errorMessage:string`。
- `m.ui.form(spec)`：`spec` 为 `table`。确认返回 `values:table`；取消返回 `nil`；创建失败
  返回 `nil, errorMessage:string`。字段 `type` 支持 `text`、`password`、`number`、
  `multiline`、`boolean`、`select`、`label`。

输入框和选择框的 `options` 字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `message` | `string` | 弹窗说明文字 |
| `multiline` | `boolean` | 是否允许多行输入，仅输入框有效 |
| `inputType` | `string` | `text`、`number` 或 `password` |
| `selectAll` | `boolean` | 获得焦点时是否全选，仅输入框有效 |
| `positiveText` | `string` | 确认按钮文字 |
| `negativeText` | `string` | 取消按钮文字 |

### HUD

```lua
local handle, errorMessage = m.hud.show("status", {
    text = "脚本正在运行",
    x = 24,
    y = 180,
    w = 260,
    h = 72,
    draggable = true,
    buttons = {
        { id = "stop", text = "停止" },
    },
})
assert(handle, errorMessage)

local event = m.hud.waitEvent("status", 30000)
if event and event.type == "click" and event.data.id == "stop" then
    print("用户点击停止")
end
m.hud.hide("status")
```

- `m.hud.show(id, spec)`：`id:any`、`spec:table`；成功返回 `handle:integer`，失败返回
  `nil, errorMessage:string`。同一 `id` 会替换旧 HUD。
- `m.hud.update(id, patch)`：`id:any`、`patch:table`；成功返回 `true`，失败返回
  `nil, errorMessage:string`。
- `m.hud.hide(id)`：`id:any`；成功关闭返回 `true`，不存在返回 `false`，失败返回
  `nil, errorMessage:string`。
- `m.hud.waitEvent(id[, timeoutMs])`：`id:any`、`timeoutMs:integer`；成功返回
  `event:table`，失败返回 `nil, errorMessage:string`。`event.type` 可以是 `click`、
  `closed`、`error` 或 `timeout`。
- `m.toast(text[, durationMs])`：`text:any`、`durationMs:integer`；成功返回
  `handle:integer`，失败返回 `nil, errorMessage:string`，不阻塞脚本。

HUD 配置字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `text` | `string` | HUD 主文字 |
| `x`、`y` | `integer` | 屏幕物理像素坐标 |
| `w`、`h` | `integer` | HUD 固定尺寸；未传时按内容自适应 |
| `gravity` | `string` | `top_left`、`center` 或 `bottom` |
| `backgroundColor`、`textColor` | `string` | `#RRGGBB` 或 `#AARRGGBB` 颜色文本 |
| `textSize` | `number` | 文字大小 |
| `alpha` | `number` | 整体透明度，范围 `0` 到 `1` |
| `padding`、`radius` | `integer` | 内边距和圆角尺寸 |
| `clickable`、`draggable` | `boolean` | 是否允许点击、拖动 |
| `durationMs` | `integer` | 自动关闭时间，`0` 或未传表示不自动关闭 |
| `buttons` | `table<table>` | 按钮数组，每项包含 `id:any` 和 `text:string` |

### HTML 页面

```lua
local page, errorMessage = m.web.open({
    file = "panel.html",
    title = "脚本面板",
    x = 100,
    y = 200,
    w = 800,
    h = 600,
})
assert(page, errorMessage)

while true do
    local event, waitError = m.web.waitEvent(page)
    if not event or event.type == "closed" then
        break
    end
    if event.type == "submit" then
        m.web.postMessage(page, { message = "Lua 已收到" })
    end
end
m.web.close(page)
```

方法返回值：

- `m.web.open(spec:table)`：成功返回 `handle:integer`，失败返回 `nil, errorMessage:string`。
- `m.web.waitEvent(handle:integer[, timeoutMs:integer])`：成功返回 `event:table`，失败返回
  `nil, errorMessage:string`；省略超时时间表示一直等待。
- `m.web.postMessage(handle:integer, data:any)`：成功返回 `true`，失败返回
  `nil, errorMessage:string`。
- `m.web.close(handle:integer)`：成功返回 `true`，失败返回 `nil, errorMessage:string`。

`m.web.open(spec)` 的 `spec` 字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `file` | `string` | 与 `url`、`html` 三选一 | 脚本目录下的相对文件名，或带协议的本地文件地址 |
| `url` | `string` | 与 `file`、`html` 三选一 | 任意 URL |
| `html` | `string` | 与 `file`、`url` 三选一 | 直接传入的 HTML 字符串 |
| `baseUrl` | `string` | 否 | `html` 的基础地址 |
| `title` | `string` | 否 | 页面 Activity 标题 |
| `x` | `integer` | 否 | 窗口左上角横坐标，单位为屏幕物理像素 |
| `y` | `integer` | 否 | 窗口左上角纵坐标，单位为屏幕物理像素 |
| `w` | `integer` | 否 | 窗口宽度，单位为屏幕物理像素 |
| `h` | `integer` | 否 | 窗口高度，单位为屏幕物理像素 |

未传 `w` 和 `h` 时全屏；只传其中一个时，另一个方向占满屏幕。

页面可直接使用：

```javascript
xiaoyv.emit("submit", { name: "Alice" });
xiaoyv.close();
xiaoyv.call("device.info", {}, "device-info");

xiaoyv.onMessage = function (data) {
  console.log(data);
};

xiaoyv.onCallResult = function (callbackId, result, error) {
  console.log(callbackId, result, error);
};
```

`xiaoyv.emit(type, data)` 将事件传给 Lua；`m.web.postMessage(handle, data)` 调用页面的
`xiaoyv.onMessage(data)` 并派发 `xiaoyv-message` 事件；`xiaoyv.call(method, params,
callbackId)` 直接调用引擎 JSON-RPC，结果通过 `xiaoyv.onCallResult` 返回。该 WebView 是
受信任脚本环境，允许本地文件、任意 URL、网络、跨域、混合内容以及 JavaScript bridge。

完整配置、事件和 C ABI 对应关系见 `docs/ANDROID_脚本_UI.md`。

## 时间

```lua
local tm = systemTime()
print(tm)

local tick = tickCount()
print(tick)
```

说明：

- `systemTime()` 无参数，返回 `integer` 类型的 Unix epoch 毫秒时间戳。
- `tickCount()` 无参数，返回 `integer` 类型的当前脚本运行毫秒数。
- 两个函数也可以通过 `m.systemTime()`、`m.tickCount()` 调用。

## 输入注入

输入注入当前只走 Root helper 常驻进程，不走无障碍，也不为每个命令拉起外部
`input` 进程。Root 不可用时返回 `false`，`touchDown` / `touchUp` 按兼容语义不返回值。

### 触摸

```lua
touchDown(0, 100, 100)
sleep(50)
local ok = touchMove(0, 200, 200)
touchUp(0)
print(ok)
```

参数：

- `id:integer`：模拟手指索引，范围 `0` 到 `4`。
- `x:integer, y:integer`：屏幕坐标。

返回：

- `touchDown`：无返回值。
- `touchMove`：返回 `boolean`，表示命令是否已执行。
- `touchUp`：无返回值。

### 按键

```lua
keyDown("home")
sleep(100)
keyUp("home")

local ok = keyPress("Back")
print(ok)
```

支持的按键标识符：

| 名称 | 标识符 | 按键码 |
|---|---|---|
| 主页键 | `Home` | `3` |

`keyDown`、`keyUp`、`keyPress` 的 `keycode` 类型均为 `string | integer`，可以传标识符，

### 文本

```lua
local ok = inputText("hello nice!!!")
print(ok)
```

`inputText(text:string)` 返回 `boolean`。它通过 Root 注入按键事件实现，适合英文、数字和
常见符号。需要完整
Unicode 文本（包括中文、Emoji 和组合文本）时使用下方的 `imeLib`。

## 输入法文本

`imeLib` 是 小鱼精灵 自己的无界面输入法。它在锁定时保存系统原默认输入法，
在解锁时恢复原输入法；高频 `setText` 直接调用活动输入法的 `InputConnection`，不
重复执行 Root 命令，也不回退到按键注入或无障碍。

```lua
local locked = imeLib.lock()
if not locked then
    print("输入法锁定失败")
    return
end

local ok = imeLib.setText("你好，小鱼精灵")
print(ok)

imeLib.unlock()
```

函数说明：

- `imeLib.lock()`：无参数；保存当前默认输入法，启用并切换到 小鱼精灵 输入法；
  返回 `boolean`。
- `imeLib.setText(text:string)`：向当前获得焦点的输入框提交完整 Unicode 文本；返回
  `boolean`。
- `imeLib.unlock()`：无参数；恢复 `lock()` 前保存的默认输入法，并禁用 小鱼精灵
  输入法；返回 `boolean`。

要求：

- `lock()` / `unlock()` 需要 Root helper 可用。
- 调用 `setText()` 前，目标输入框必须已经获得焦点。
- `setText()` 前必须成功调用过 `lock()`；调用结束后应调用 `unlock()` 恢复用户原输入法。
- 自有 API 也可用 `m.ime.lock()`、`m.ime.setText(text)`、`m.ime.unlock()`。

### 运行环境

```lua
print(getRunEnvType())
```

当前返回：

- `getRunEnvType()` 无参数，返回类型为 `string`。
- `root`：Root helper 已就绪。
- `none`：当前没有可用 Root 输入注入环境。

## 截图返回值

```lua
local w, h, pixels = m.capture()
if not w then
    print("截图失败：", h)
    return
end

print("宽 =", w)
print("高 =", h)
print("点阵地址 =", string.format("0x%x", pixels))
```

成功时：

- 第 1 个返回值 `w:integer`：宽度。
- 第 2 个返回值 `h:integer`：高度。
- 第 3 个返回值 `pixels:integer`：点阵首地址。
- 点阵格式固定为紧凑 RGBA，长度为 `w * h * 4`。

失败时：

- 第 1 个返回值：`nil`。
- 第 2 个返回值 `errorMessage:string`：错误信息。

## 缓存规则

- `m.keepCapture()` 无参数，返回 `true:boolean`。
- `m.releaseCapture()` 无参数，返回 `true:boolean`。
- `m.setCaptureCacheMs(ms:integer)` 成功返回设置后的 `ms:integer`，失败返回
  `nil, errorMessage:string`。
- 默认缓存时间：`20ms`。
- 缓存时间内重复调用 `m.capture()`，复用当前点阵。
- 超过缓存时间后重新截图并覆盖内部缓存。
- `m.keepCapture()` 后一直复用当前帧。
- `m.releaseCapture()` 后恢复按时间缓存。
- 脚本运行中不会主动清空截图缓存；脚本结束后由引擎统一释放。

## 找色

```lua
local x, y = m.findColors(0, 0, 1079, 1919, 2, 0x101010, "0|0|FFFFFF,10|5|FF0000-101010")
if not x then
    print("找色失败：", y)
    return
end

print("找到：", x, y)
```

参数：

- `x1:integer, y1:integer, x2:integer, y2:integer`：查找范围。
- `dir:integer`：扫描方向，取值 `1` 到 `8`，沿用旧找色算法。
- `sim:integer`：默认容差，格式为 `0xRRGGBB`。
- `colors:string`：多点颜色字符串，格式为 `x|y|RRGGBB`，可用 `-RRGGBB` 给单点设置
  独立容差。

返回：

- 找到：返回 `x:integer, y:integer`。
- 未找到或失败：返回 `nil, errorMessage:string`。

说明：

- 找色直接使用当前截图缓存，不需要“是否截屏”参数。
- 如果要固定同一帧连续找色，先调用 `m.keepCapture()`；完成后调用 `m.releaseCapture()`。
