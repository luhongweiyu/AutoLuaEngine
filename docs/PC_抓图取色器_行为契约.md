# PC 抓图取色器行为契约

本文档记录抓图取色器重构后的最终工作逻辑。实现、测试和界面文案必须与这里保持一致。

## 图片与标签

- 从磁盘打开的图片属于“文件图片”；设备截图、浮动抓图、窗口截图、拖入图片数据和裁剪结果属于“临时图片”。
- 没有文件名的临时图片统一使用 `yyMMdd_HHmmss` 命名，例如 `260718_122656`。
- 关闭单个临时图片标签时直接关闭，不弹保存提示。大量抓图时，关闭标签必须是低成本操作。
- 关闭已修改的文件图片标签时询问保存，避免覆盖式编辑丢失。
- 退出整个程序时，如果仍有未保存的临时图片或已修改文件图片，只汇总询问一次；不逐个弹窗打断用户。
- 临时图片保存后转为文件图片；图片标签必须真实反映未保存/已修改状态，不能为了减少弹窗伪装成已保存。
- 图片撤销历史同时受数量和内存预算限制，不能因连续二值化或旋转无限占用内存。
- 取色点、范围、二值化参数、点阵提取结果、文字和当前选择属于当前工具会话；切换图片时原样带到下一张，不按图片分别保存，也不得清空或等待恢复。

## 取色与框选

- 取色、刷新、删除、启用、偏色和基准点在工具会话中连续使用；切换图片后仍显示上一张图片离开时的内容。
- 旋转、翻转、二值化不会静默重写已有取色点；用户通过“刷新选中/刷新全部”明确重新取样。
- 框选范围由取色和点阵字库共享，并在图片标签之间连续沿用；没有图片时仍保留给下一张图片。
- 当前会话范围由 `ImageWorkspace` 在活动图片、下一张图片和各面板之间连续传递；两个面板仅显示并提交输入，不保存第二份范围。
- 高频取色不增加确认弹窗。无图片、无有效点或格式错误时，在操作发生时给出明确提示。
- RGB 和偏色支持双击修改；单击颜色块打开颜色选择器。选中整行时颜色块仍须显示真实颜色。

## 二值化与偏色

- 只有“二值化 / 偏色”标签可见时才显示实时预览；参数变化立即刷新，离开标签或隐藏图像工具时立即恢复原图显示。
- 颜色模式与点阵提取共用 `RRGGBB-偏色|...` 规则格式；启用的取色点变化后同步规则，仅在二值化标签可见时重新预览，规则框也允许手动修改。
- 界面不提供“预览”“实时预览”或“还原”按钮；参数变化和离开标签都会使旧异步结果失效，旧任务完成后不得回写。
- 没有有效预览时点击“应用”，先计算最新结果，完成后再应用；不能静默无效，也不能应用上一轮结果。
- 大图计算在工作线程执行；同一时间只运行一个任务，连续修改参数只保留最后一次请求。

## 点阵字库

- 同一文字允许保存多个不同点阵，用于不同字体、背景和显示状态。
- 同一点阵只能对应一个文字；同一文字与同一点阵不重复保存。
- 点阵冲突只在实际发生时询问，不要求用户提前选择策略。
- 批量加入在临时副本中完成；取消整批操作时不得留下半批新增或替换。
- “替换选中”也必须遵守点阵唯一性，不能绕过冲突规则制造歧义字库。
- 字库新增、替换和删除产生未保存状态。打开其他字库或退出程序前必须处理未保存修改。
- 字库列表直接绑定 `FontDictionaryDocument`，不得再由面板复制记录后手工重建第二份列表。
- 字库文件选择、未保存确认、冲突处理和保存按钮只属于 `FontDictionaryEditor`；`FontPanel` 只提供当前点阵和批量点阵。
- 保存到已有文件前必须读取文件当前内容并合并外部新增记录；冲突必须明确处理，不能直接截断覆盖。
- 保存使用临时文件和原子提交，失败时保留原文件。
- 点阵提取第一行固定为未分割点阵，后续行是自动分割结果；即使只有一个字形且点阵相同，也必须保留对应的分割行。
- 提取表格固定为“序、文字、宽高、点数”；默认选中首个分割结果，未分割点阵由用户主动选择后再显示。
- 重新提取时先清空旧结果；新提取失败不得继续显示上一轮点阵。切换图片不清空、不切换另一份结果。

## 设备连接与测试

- ADB 转发建立、图片编码和网络请求不能阻塞界面线程。
- 普通请求必须有超时；设备截图、投影和主动刷新连接状态在执行期间防止重复提交。
- 测试脚本运行时持续读取日志，并提供停止操作；脚本运行请求本身不使用普通短超时。
- 停止请求被设备接受后，停止按钮保持禁用，直到原 `script.run` 请求真正结束，避免重复发送停止命令。
- 启动和修改连接设置后自动检查一次状态；其他操作直接执行，不在命令前重复探测。
- 收到任意 HTTP 响应或发生传输失败时自动更新连接状态；HTTP/JSON-RPC 业务错误不得误判为断线，ADB 转发失效后允许重新建立。
- 本工具创建过的每个 ADB 动态转发端口都必须被记录，切换设置或退出时逐个移除，不能因端点失效丢失清理信息。
- 多个客户端可同时连接设备，但本工具内部同类操作按明确状态串行化，返回结果不得互相覆盖。
- `DeviceTransport` 只处理端点和协议；截图、投影、脚本状态只属于 `DeviceClient`，两层不得互相复制状态。

## 界面与设置

- 保存主窗口尺寸、位置、停靠布局、面板显隐和主题，下次启动恢复。
- 工具栏顺序、图标和动作语义只维护一份配置。
- 高频按钮使用禁用/状态变化表达运行状态；只有需要用户决策时才弹窗。
- 自定义生成格式加载失败必须可见，不能静默消失；生成器执行需要时间限制或可中断边界。
- 内置格式和用户格式统一放在 exe 同级的 `formats` 目录；打开格式目录、重载和生成都读取这一处，内置格式允许用户直接修改。

## 验证要求

- 图片、字库和设备协议的不变量必须有自动测试，不能只依赖人工点击。
- 异步预览必须测试图片版本变化后的旧结果丢弃。
- 设备测试至少覆盖健康检查、原始截图帧校验和重复请求防抖。
- 每次完整构建都运行 CTest；界面结构变化后还需启动程序检查暗色/亮色主题、停靠布局、
  文本溢出和控件遮挡。
