# 构建与运行

## 构建 APK

```powershell
$env:JAVA_HOME='D:\soft\Android\Android Studio\jbr'
$env:PATH="$env:JAVA_HOME\bin;$env:PATH"
.\gradlew.bat :app:assembleDebug
```

工作目录：

```text
engines/android
```

## 脚本目录

Android 脚本默认存放在：

```text
/sdcard/xiaoyv/scripts/
```

首次启动会申请脚本存储访问：Android 11 及以上打开“所有文件访问”系统页，Android 10
及以下申请传统读写存储权限。授权后创建默认目录，并且只在当前安装首次初始化时复制内置
示例，不覆盖已有文件，用户之后删除示例不会再次生成。用户可在 App 设置页直接输入
`/sdcard` 下的其他脚本目录。脚本列表、悬浮窗运行入口和引擎读取的都是当前配置目录中的
真实文件，外部编辑器保存后返回 App 即可刷新。

## 加密脚本包

脚本项目根目录使用 `alpkg.json` 指定入口，`entry` 必填，`exclude` 可选：

```json
{
  "entry": "main.lua",
  "exclude": [".git/", ".vscode/", "dist/", "*.bak"]
}
```

首次打包时若缺少 `alpkg.json`，打包器会自动创建这份默认配置后继续执行；若项目中没有
`main.lua`，会在入口校验阶段明确报错。

执行 `tools/pack/构建打包器.ps1 -Release` 后，可使用：

```powershell
.\tools\pack\打包脚本包.ps1 -ProjectDirectory 'E:\我的脚本项目'
```

输出为 `项目目录\dist\项目名.alpkg`。App 会在脚本列表显示并可直接运行 `.alpkg`：Lua
文件是去调试信息的 Lua 5.4.8 字节码，经 XChaCha20-Poly1305 认证加密后由 `libengine.so`
从 ZIP 内存加载；HTML、图片和其他资源保持原格式，直接从包内读取，不解压到磁盘。
包内 HTML 的相对 CSS、JavaScript、图片等资源会自动从同一个 `.alpkg` 读取。
脚本需要读取 JSON、CSS、图片等原始资源时，可调用
`m.read_alpkg_file("项目相对路径")`；它只读取已打包资源，不回退到包外文件。
完整格式见 [ALPKG 格式](../shared/package/ALPKG_格式.md)。
