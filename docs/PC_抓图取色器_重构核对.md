# PC 抓图取色器重构核对

本文档用于防止后续开发重新回到“在旧代码上继续补丁”的方式。每次修改工作流时，应先更新
[行为契约](PC_抓图取色器_行为契约.md)，再修改唯一负责该状态的模块和对应测试。

## 重构方式

- `ide/xiaoyv-tools` 曾清空后重建，但“重新建文件”不作为重构完成的证明。
- Git 历史只用于核对曾经存在的功能；验收依据是当前职责边界、工作流和测试，不依据新增/删除文件的显示方式。
- 图片、取色点、字库、异步分析、设备连接分别只有一个状态所有者。
- 主窗口只做模块协调，不保存业务状态副本。
- 菜单和工具栏共用一个动作目录，不再手工维护两份顺序和图标。
- 测试直接针对行为契约，不以旧实现细节作为通过条件。

## 代码量审计

- 重构前生产源码为 32 个文件、6282 行。
- 2026-07-18 完成边界重写后生产源码为 52 个文件、7825 行，比旧版多 1543 行。
- 测试源码由旧版 769 行调整为 5 组共 1070 行，恢复并覆盖取色、基准点、键盘采集、放大镜、二值化控件、点阵编辑、设备空回调、投影协议、脚本停止和源码架构边界。
- 生产代码增量来自显式的 `ImageWorkspace`、`DeviceTransport`、`FontDictionaryEditor` 和 `ScriptEditorPanel` 边界；旧大类中的对应实现已经删除，不是新旧两套并存。
- 行数增加不是质量证明，行数减少也不是目标；禁止用压缩排版、机械拆文件或删除必要测试制造“变少”的结果。
- 每轮应继续检查重复状态、重复 UI 重建、字符串协议键和无法测试的隐式工作流。

## 旧问题处理

| 旧问题 | 新实现 |
|---|---|
| 临时截图关闭时反复询问保存 | 单标签直接关闭；只在退出程序时汇总提醒 |
| 图片标签为了少弹窗伪装成已保存 | 临时和修改状态真实显示 `*` |
| 异步二值化可能把旧结果写入新标签 | 文档、图片版本、参数、请求号四项共同校验 |
| 点击“应用”可能应用上一轮预览 | 无有效预览时重新计算最新参数后应用 |
| 字库用字符串数组临时判重 | 独立 `FontDictionaryDocument` 强制点阵唯一 |
| 字库面板手工复制并重建记录列表 | `FontDictionaryDocument` 直接作为 Qt 列表模型 |
| `FontPanel` 同时处理提取、字库文件和冲突弹窗 | 提取留在 `FontPanel`，字库事务由 `FontDictionaryEditor` 独占 |
| 同一个文字不能保存多种点阵 | 文字不做唯一键，同文字不同点阵可并存 |
| 同一点阵可对应多个文字 | 发现冲突时跳过、替换或取消，不允许静默新增 |
| 批量入库中途取消仍残留前半批 | 临时副本完成整批，确认后一次提交 |
| 保存字库直接截断原文件 | 保存前重读、合并外部内容、`QSaveFile` 原子提交 |
| ADB 首次连接阻塞界面 | 异步 `QProcess`，等待操作共用一次转发建立 |
| JSON-RPC 回调被重复 move 后丢失 | 成功和失败路径各持有可调用副本 |
| 截图、投影可被连续重复提交 | 每类操作独立忙状态，界面同步禁用 |
| 设备忙状态依赖字符串键 | 使用固定 `DeviceClient::Operation` 枚举 |
| `DeviceClient` 同时维护 ADB、HTTP 和脚本会话 | `DeviceTransport` 负责传输，`DeviceClient` 负责设备业务 |
| ADB 端点失效后遗失旧动态端口 | 记录本进程创建的全部端口，退出或切换设置时精确清理 |
| 脚本测试结束后才一次显示日志 | 运行中持续拉取，停止和最终日志有明确状态 |
| 停止请求返回后可再次重复点击 | 保持停止忙状态，直到脚本运行请求真正结束 |
| 工具栏创建和刷新图标维护两份列表 | `action_catalog.h` 是唯一动作目录 |
| 主窗口保存图片标签、待用范围和保存关闭流程 | `ImageWorkspace` 独占图片工作区状态 |
| 高倍率像素网格遍历整张大图 | 只绘制当前可见像素范围 |
| 方向键受 DPI 舍入影响跳到错误方向 | 使用图片逻辑坐标移动，并忽略程序鼠标移动回灌事件 |
| 自定义格式错误静默消失或无限运行 | 加载错误可见，文件大小和执行时间有边界 |

- [x] 取色只能由回车/空格触发，方向键按图片坐标逐像素移动。

- [x] 二值化支持颜色、偏色、灰度、反色、参数变化自动预览、还原和应用。

- [x] 浮动区域抓图和 Windows 拖动选窗抓图可用。
- [x] LAN/ADB、设备截图、投影、脚本测试、停止和实时日志可用。
- [x] 暗色/亮色等多主题、高 DPI 图标和紧凑停靠窗可用。
- [x] 主窗口状态、布局和面板显隐可持久化。
- [x] 模型、界面、设备协议和主窗口测试已建立。
- [x] 面板只提交框选输入、脚本停止防重复和字库模型视图已加入自动测试。
- [x] 图片工作区、脚本编辑器和字库编辑器的独立行为已加入自动测试。
- [x] CTest 增加源码架构规则，禁止旧职责重新堆回主窗口、设备客户端和点阵面板。

## 后续修改规则

1. 先判断需求属于哪个状态所有者，不在主窗口复制状态。
2. 改行为前先更新行为契约；纯视觉修改不改变业务契约。
3. 同一规则只能在模型或服务中实现一次，面板只负责用户输入和结果展示。
4. 高频操作不增加确认框；只有不可逆冲突和退出丢失才要求用户决策。
5. 新增异步操作必须定义取消、超时、旧结果失效和重复提交行为。
6. 新增文件必须在文件头写明用途，并加入对应构建目标和测试。
