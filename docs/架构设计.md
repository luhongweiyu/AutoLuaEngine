# 架构设计

## 当前分层

```text
App/UI
  负责脚本选择、状态展示、设置和悬浮控制

EngineService (:engine)
  负责脚本运行、停止、状态广播和 HTTP 控制入口

libengine.so
  负责 Lua Runtime、core/api 统一核心实现、C ABI 门面和当前截图核心
```


## Lua 绑定

Lua HostApi 位于：

```text
engines/android/app/src/main/cpp/runtime/lua/host_api.cpp
engines/android/app/src/main/assets/runtime/api_m.lua
```

当前 Lua HostApi 绑定以下脚本函数，但只做参数转换和返回值封装：

```text
print(...)
m.sleep(ms)
m.log.print(text)
m.capture()
m.keepCapture()
m.releaseCapture()
m.setCaptureCacheMs(ms)
m.findColors(x1, y1, x2, y2, dir, sim, colors)
```

实际逻辑统一在 `core/api`，Lua HostApi 通过 `system_c_api` 的 C ABI 调用它们。

## 当前边界规则

- App/UI 只负责界面、脚本文件选择、悬浮窗和状态展示。
- EngineService 只负责引擎进程、HTTP 服务、脚本控制和状态广播。
- 所有脚本 API 的真实逻辑先进入 `libengine.so/core/api`，语言绑定层只做转换。
- `system_c_api` 是 C ABI 门面，不承载具体业务逻辑。
- Java 桥只处理 Android 必须由 Java 完成的能力，例如 Root helper 启动和平台桥接。
