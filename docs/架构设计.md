# 架构设计

## 当前分层

```text
App/UI
  负责脚本选择、状态展示、设置和悬浮控制

EngineService (:engine)
  负责脚本运行、停止、状态广播和 HTTP 控制入口

libengine.so
  负责 Lua Runtime、日志缓冲、C ABI 和当前截图核心
```

## 当前 native 边界

统一 C ABI 位于：

```text
engines/android/app/src/main/cpp/core/system_c_api.h
engines/android/app/src/main/cpp/core/system_c_api.cpp
```

当前导出：

```c
engine_version()
engine_capabilities_json()
screen_capture()
screen_keep_capture()
screen_release_capture()
screen_set_capture_cache_ms()
screen_clear_capture_cache()
screen_last_error()
```

## Lua 绑定

Lua HostApi 位于：

```text
engines/android/app/src/main/cpp/runtime/lua/host_api.cpp
engines/android/app/src/main/assets/runtime/api_m.lua
```

当前只绑定最小截图能力。

## 当前边界规则

- App/UI 只负责界面、脚本文件选择、悬浮窗和状态展示。
- EngineService 只负责引擎进程、HTTP 服务、脚本控制和状态广播。
- 系统能力统一从 `libengine.so` 的 C ABI 增加，不再把业务 API 分散到 Java 桥或 HTTP 命令里。
- 当前系统能力只有 Root 截图核心。
