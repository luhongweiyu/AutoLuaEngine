# 架构设计

## 当前分层

```text
App/UI
  负责脚本选择、状态展示、设置、悬浮控制、脚本 UI 绘制和 RootDaemon 生命周期

RootDaemon（uid=0）
  负责 Root 截图、输入注入和输入法系统切换，只接受本应用认证的回环 socket 请求

EngineService (:engine)
  负责脚本运行、停止、状态广播、HTTP 控制入口和到 RootDaemon 的客户端会话

libengine.so
  负责 Lua Runtime、core/api 统一核心实现、C ABI 门面和当前截图核心
```

## 当前边界规则

- App/UI 只负责界面、脚本文件选择、悬浮窗和状态展示。
- EngineService 只负责引擎进程、HTTP 服务、脚本控制和状态广播。
- 所有脚本 API 的真实逻辑先进入 `libengine.so/core/api`，语言绑定层只做转换。
- `system_c_api` 是 C ABI 门面，不承载具体业务逻辑。
- Java 桥只处理 Android 必须由 Java 完成的能力，例如 RootDaemon 生命周期、认证 socket 和平台桥接。
- `:engine` 不持有 `su` 会话；强停它不会关闭 RootDaemon，也不会在下一次脚本运行时重新请求 Root 授权。

## 脚本 UI 路径

```text
Lua HostApi
  -> system_c_api / core/api/ui_api
  -> AndroidBridge JNI（:engine）
  -> App 主进程的 Dialog Activity、HUD Service、WebView Activity
  -> EngineHttpServer 的 ui.event
  -> core/api/ui_api 会话队列
  -> Lua m.ui.waitEvent / m.hud.waitEvent / m.web.waitEvent
```

`libengine.so` 负责会话 ID、配置、事件队列、等待和 C ABI；App 主进程只负责 Android
View、窗口和 WebView。三个界面实现分别是：

- `ScriptDialogActivity`：原生提示、确认、输入、选择和表单。
- `ScriptHudService`：脚本 HUD 悬浮层、按钮、拖动和定时关闭。
- `ScriptWebActivity`：HTML、WebView 和 JavaScript bridge。

脚本 UI 事件不会在 Android UI 线程执行 Lua。事件先通过本机 HTTP `ui.event` 写入
native 队列，再由脚本线程读取。完整 API 见 `docs/ANDROID_脚本_UI.md`。
