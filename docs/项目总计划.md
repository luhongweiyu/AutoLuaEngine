# 项目总计划

## 当前目标

第一阶段只做 Android + Lua，并把脚本 API 的真实逻辑收敛到 `libengine.so/core/api`。

当前已经保留：

- App/UI
- EngineService 独立进程
- HTTP 控制通讯
- Lua 脚本运行、停止、日志
- 运行时 C ABI：print / log / sleep
- 截图 C ABI：capture / cache / keep / release
- 找色 C ABI：find
- 插件 so 函数表：engine_get_api

已实现能力走 `core/api -> system_c_api -> Lua HostApi`。未实现的能力不保留对外函数。

## 当前 C ABI

```c
int runtime_print(const char* text);
int runtime_log_print(const char* text);
int runtime_sleep(int durationMs);
int runtime_sleep_interruptible(...);
int screen_capture(int* width, int* height, unsigned char** pixels);
void screen_keep_capture();
void screen_release_capture();
int screen_set_capture_cache_ms(int durationMs);
void screen_clear_capture_cache();
const char* screen_last_error();
int engine_findColors(...);
const char* engine_findColorsLastError();
const EngineApi* engine_get_api();
```

## 当前 Lua API

```lua
print(...)
m.sleep(ms)
m.log.print(text)
m.capture()
m.keepCapture()
m.releaseCapture()
m.setCaptureCacheMs(ms)
m.findColors(x1, y1, x2, y2, dir, sim, colors)
```

## 下一步原则

新增脚本 API 时先做 `core/api` 真实现，再做 C ABI 门面，最后接 Lua / JS / Go。
语言绑定层不写最终逻辑。

每个能力一次只做一个清晰闭环：

1. `core/api` 核心实现。
2. Android/Root helper 桥接。
3. C ABI 门面。
4. Lua 绑定。
5. 文档。
6. 示例脚本。

未完成闭环的能力不写入对外 API。
