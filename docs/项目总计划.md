# 小鱼精灵项目总计划

## 当前目标

第一阶段只做 Android + Lua。固定脚本 API 的真实逻辑收敛到
`libengine.so/core/api`，动态 Java 对象能力由 `libengine.so/runtime/lua` 接入。

当前已经保留：

- App/UI
- EngineService 独立进程
- HTTP 控制通讯
- Lua 脚本运行、停止、日志
- 运行时 C ABI：print / log / sleep / systemTime / tickCount
- 屏幕点阵 C ABI：物理截图、图片屏幕、cache / keep / release
- 找色 C ABI：find
- 图像 C ABI：保存截图、模板找图、模板缓存
- OCR C ABI：内置/自定义 RapidOCR ONNX 模型加载、释放、识别、找文字
- 点阵字库 C ABI：任意宽高完整点阵、11 位候选索引、完整识字/找字、目标字形快速找字
- 输入 C ABI：touch / key / inputText
- 输入法 C ABI：imeLock / imeSetText / imeUnlock
- 脚本 UI C ABI：dialog / hud / web 会话、事件和关闭
- 设备 C ABI：应用状态、硬件信息、显示信息、应用管理、Root exec 和系统控制
- ALPKG 资源 C ABI：readAlpkgFile，供 Lua / JS / Go / 插件读取当前包内原始资源
- 插件 so 函数表：engine_getApi
- Lua-Java 动态互操作：import / Java 对象 / 接口回调 / LuaEngine 兼容类
- Lua native 多线程：共享 VM、真实工作线程、VM Gate、beginThread / Thread.newThread
- ALPKG 加密脚本包：ZIP 容器、Lua 去调试字节码、XChaCha20-Poly1305、包内 require / WebView 资源

固定 API 走 `core/api -> system_c_api -> Lua HostApi`；Java 动态调用走
`Lua Java Bridge -> JNI -> JavaInteropBridge`。未实现的能力不保留对外函数。

## 下一步原则

新增语言无关设备 API 时先做 `core/api` 真实现，再做 C ABI 门面，最后接 Lua / JS / Go。
语言绑定层不写最终设备逻辑。

动态对象桥是例外：Java 的任意类和方法无法预先定义固定 C ABI，统一反射、重载和
类型转换放在 `JavaInteropBridge`，各语言运行时只实现自己的对象包装与回调接入。
语言自身的并发模型也是例外：Lua 使用 native worker + VM Gate，JS 使用事件循环，
Go 使用 goroutine，三者只在调用设备 API 时汇合到 `core/api`。

每个能力一次只做一个清晰闭环：

1. `core/api` 核心实现。
2. Android/Root helper 桥接。
3. C ABI 门面。
4. Lua 绑定。
5. 文档。
6. 示例脚本。

未完成闭环的能力不写入对外 API。
